Logic

2

触发分享行为 用户可保存到本地，也可复制文案，后续扩展朋友圈、微信聊天、小红书分享。
技术建议：优先服务端生成海报，减少前端机型兼容与字体排版问题。

Implementation Notes

技术实现建议与数据边界

给研发评审时可直接用来拆模块、定表结构和接口边界。

数据快照
服务端海报
配置化系统
Data Model
关键实体
  • 活动表：活动标题、banner、规则、顾问二维码、状态。
  • 书籍表：书名、封面、描述。
  • 题目表：题型、题干、选项、答案、解析、Day、排序。
  • 用户打卡计划表：参与时间、连续天数、状态。
  • 用户打卡记录表：答案快照、图片 URL、得分、提交时间。
  • 福利配置表与用户福利状态表。
  • 金句表与海报模板关联表。
API Scope
核心接口
  • 获取活动首页信息。
  • 创建或读取用户 21 天打卡计划。
  • 获取当日打卡内容。
  • 提交答案、上传图片、提交打卡。
  • 获取金句海报信息或海报 URL。
  • 获取福利列表与解锁状态。
  • 获取结营测评题与提交结果。
  • 后台活动、内容、福利、海报配置接口。
Risk Control
注意事项
  • 题目与测评结果要保存快照，避免后续修改污染历史数据。
  • 海报建议服务端生成并缓存，减少前端兼容问题。
  • 上传图片需校验格式和大小，必要时接内容安全审核。
  • 后台需有权限和操作日志，防止误发布。
  • 同一用户同一天仅记一次有效打卡。
  • 发布前必须做内容完整性校验。

活动规则

  • 采用连续 21 天打卡机制，中断后需重新累计连续天数。
  • 每日习题为必做，拍照入口根据活动配置决定是否展示。
  • 当日首次提交成功记为有效打卡，并触发金句海报。
  • 达到指定连续打卡天数后自动解锁对应福利。
  • 连续完成 21 天后解锁结营测评与结果页。
